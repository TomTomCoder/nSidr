# Teable — Standalone Deployment

Single-host setup: Postgres + Redis + Teable app, all in one `docker compose`.

## Quickstart (recommended)

```bash
./setup.sh
```

The script will:
1. Check prerequisites (Docker, Docker Compose, curl)
2. Ask for your public URL and optional AI provider
3. **Generate random secrets** for Postgres and Redis
4. Write `.env`, pull images, start services
5. Wait for the health check and print the URL

Re-run `./setup.sh` at any time — it detects an existing `.env` and lets you keep or reconfigure it.

## Upgrade

```bash
./update.sh            # pull latest and restart the app container
./update.sh --tag v1.2 # pin to a specific version
```

The upgrade script pulls the new image, restarts only the `teable` container (Postgres and Redis are untouched), and waits for the health check before declaring success.

## Manual setup (advanced)

```bash
cp .env .env.local     # edit passwords, PUBLIC_ORIGIN, AI keys
docker compose up -d
```

## Key variables

| Variable | Description |
|---|---|
| `PUBLIC_ORIGIN` | Full URL users will access (e.g. `https://teable.example.com`) |
| `POSTGRES_PASSWORD` | Auto-generated by `setup.sh` — change if needed |
| `REDIS_PASSWORD` | Auto-generated by `setup.sh` — change if needed |
| `AI_PROVIDER` | `openai` \| `anthropic` \| `openai_compatible` — enables AI chat |
| `OPENAI_API_KEY` | Required when `AI_PROVIDER=openai` |
| `ANTHROPIC_API_KEY` | Required when `AI_PROVIDER=anthropic` |
| `AI_MODEL` | e.g. `gpt-4o` or `claude-3-5-sonnet-20241022` |

## Services

| Service | Port | Notes |
|---|---|---|
| Teable app | `3000` | Main UI + API |
| Postgres | `42345` | Exposed for DB proxy connections |
| Redis | internal | Not exposed externally |

## Useful commands

```bash
# View logs
docker compose logs -f teable

# Stop everything
docker compose down

# Full reset (destroys all data)
docker compose down -v
```
