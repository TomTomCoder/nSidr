<objective>
Add the single-cell AI regenerate endpoint per D-16-02 + D-16-03. The controller
method is a thin handler: load the field via FieldService, build the prompt the
same way the bulk auto-fill pipeline does (source field values for that row),
delegate to `AiService.generateForField` (built in 16-02), then write the
validated result through the existing `RecordOpenApiService.updateRecord` path
so collaborators see the cell update via the normal op-event stream. Validation
failures surface to the caller as `{ validated:false, error }` — never throw,
never silently null the value via a side channel.

Purpose: Plan 16-04 (frontend menu item) calls this endpoint; the existing bulk
pipeline can later migrate to the same `generateForField` underneath without
touching this surface.
Output: 1 openapi route + client, 1 controller method, 1 service method, vitest.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → API | Authenticated user requests cell regeneration (record|update gate) |
| API → LLM gateway | Backend calls AiService.generateForField — single ingress |
| LLM → typed cell | Validated value crosses into typed storage via updateRecord |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-06 | Spoofing | regenerate endpoint | mitigate | @Permissions('record|update') guard — same gate as cell edit |
| T-16-07 | Tampering | LLM output → cell write | mitigate | generateForField validates (16-02); on validated:false the write path is NOT invoked |
| T-16-08 | Repudiation | Silent cell write bypassing op-events | mitigate | Write via this.updateRecord (existing op-event path) — NO direct prisma.record.update |
| T-16-09 | Information disclosure | Error message echoes Zod detail | accept | Error already authenticated; no PII in schema names |
| T-16-10 | Denial of service | Regenerate spam | accept | Per-request cost is single LLM call; rate-limiting handled at gateway/account tier |
| T-16-11 | Elevation of privilege | Non-Ai field regenerate | mitigate | FieldType.Ai check throws BadRequestException before any LLM call |
</threat_model>
