<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser UI → API | User-initiated POST carrying session credentials |
| Mutation hook → cache | No optimistic writes — op-event is single source of truth |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-12 | Spoofing | Regenerate action by unauthorized user | mitigate | Server-side @Permissions('record|update') gate (16-03); UI also gates on canUpdate but server is authoritative |
| T-16-13 | Tampering | UI hiding "validated:false" error from user | mitigate | toast.error fires on every validated:false response; no silent swallow |
| T-16-14 | Repudiation | Optimistic cache write desyncs from server | mitigate | Hook does NOT write cache optimistically — op-event is the only write path |
| T-16-15 | Information disclosure | Server error message rendered in toast | accept | Message is the server's Zod validation summary; no sensitive payload |
</threat_model>
