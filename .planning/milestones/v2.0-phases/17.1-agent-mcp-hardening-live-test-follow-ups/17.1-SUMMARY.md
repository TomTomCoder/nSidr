## Lessons learned (anti-patterns to log)

1. **Unit-test mocks must match real production shape, not the convenient shape the consumer expects.** Bug-3-redux happened because both the mapper and its mock used the wrong shape — drift was invisible. Anti-pattern catchphrase: *"mock-shape drift."* Pattern fix: bind mocks to real type constructors (`mkInterfaceToolDef` ensures any future change to `ToolDefinition` breaks the mock at compile time).

2. **Unit-passing ≠ runtime-correct.** Same lesson as Phase 17. The drift-resistant mock helper is a partial mitigation; the deeper fix is live-UAT gates on every phase that ships an integration surface.

3. **DI bug-1 is contagious:** the original `import type` lesson (2026-06-06 commit 00450e8b5) needed reinforcement — this phase added `InterfaceToolsService` as a VALUE import (correct) but a future engineer might add another DI'd service and reach for `import type` again. Consider eslint rule.
