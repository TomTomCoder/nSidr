## Task 1 — Cross-cutting test suite

Create `apps/nestjs-backend/src/features/plugin/__tests__/extension-system.spec.ts`:

Test cases:
1. **install-then-gate**: call `installByUrl` with a mocked fetch returning a valid tools/list response.
   Assert Plugin created with `isExtension=true, consentedAt=null`.
   Call `discoverPluginTools(baseId)` mock. Assert the unconsented extension NOT in results.

2. **install-then-consent-then-discover**: same setup, then call `consentExtension`.
   Assert `consentedAt` is set. Assert `discoverPluginTools` now INCLUDES the extension's tools.

3. **ssrf-block**: `installByUrl` with a URL that SsrfGuardService will block.
   Assert throws with VALIDATION_ERROR. Assert no Plugin record created.

4. **manifest-whitelist**: fetch returns `{name: "tool", description: "d", inputSchema: {...}, malicious: {__proto__: ...}}`.
   Assert stored `toolManifest` does NOT contain `malicious` key.

5. **revoke-consent**: consent, then revoke. Assert `consentedAt` is null again and discovery
   excludes the extension.

Use vitest `vi.mock` for fetch + SsrfGuardService.
