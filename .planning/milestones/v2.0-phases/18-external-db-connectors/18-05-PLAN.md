<objective>
EXTDB-02 surface — the largest/new piece. Map introspected external Postgres tables (18-04) to
READ-ONLY virtual tables in Teable and render them browsable. Query-through only: no row copies,
no Teable write paths touch them. Confirmed during planning: Teable has NO existing virtual/external
table concept (real tables map via dbTableName), so this is a genuinely new layer.

Purpose: Federated read access to external Postgres without import.
Output: virtual-table service + type mapper + read API + read-only grid UI.
</objective>
