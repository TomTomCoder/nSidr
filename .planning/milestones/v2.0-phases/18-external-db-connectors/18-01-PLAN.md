  <files>apps/nestjs-backend/src/features/external-connection/ssrf-guard.service.ts, apps/nestjs-backend/src/features/external-connection/ssrf-guard.service.spec.ts</files>
  <read_first>none — new isolated module; node dns + net are the only deps</read_first>
  <behavior>
    - assertHostAllowed('169.254.169.254') rejects (cloud metadata)
    - assertHostAllowed('localhost'/'127.0.0.1'/'::1') rejects (loopback)
    - assertHostAllowed('10.x'/'192.168.x'/'172.16-31.x'/'fc00::/7') rejects (private)
    - assertHostAllowed('169.254.x') rejects (link-local)
    - DNS rebinding: a hostname resolving to a private IP rejects (resolve THEN check every returned address)
    - optional allowlist: if EXTERNAL_DB_HOST_ALLOWLIST set, only listed hosts pass
    - a public host (or allowlisted host) resolving to public IPs passes
  </behavior>
  <action>Implement SsrfGuardService.assertHostAllowed(host): resolve the host with dns.promises
  to ALL addresses, then reject if ANY resolved address falls in loopback, private (RFC1918 +
  fc00::/7), link-local (169.254/16, fe80::/10), or the cloud-metadata address 169.254.169.254.
  Support an optional allowlist via env (EXTERNAL_DB_HOST_ALLOWLIST, comma-separated). Throw a
  typed error (e.g. SsrfBlockedError) on rejection. This MUST be callable standalone so every
  connect path can gate through it. No fenced implementation here — define the method signature
  and the rejection rules above.</action>
