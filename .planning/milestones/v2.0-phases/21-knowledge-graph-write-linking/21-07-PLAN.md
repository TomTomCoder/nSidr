    4) **Get links**: Agent prompt → "list the links from the new doc"
       Expected tool: `get_doc_links`. Response includes one outgoing link with label='references'.

    5) **Traversal**: Agent prompt → "search the knowledge base for '<term in the LINKED doc, not the new one>' and follow links 1 hop"

    8) **Hop-cap negative**: search_knowledge_base with `maxHops: 4`. Expect BadRequestException. Record.

    9) **UI sanity**: open the new doc in the doc viewer; the "Linked docs" sidebar shows one outgoing entry to the linked doc; click it and the viewer navigates.
