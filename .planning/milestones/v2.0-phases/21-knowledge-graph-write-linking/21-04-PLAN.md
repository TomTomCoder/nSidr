`buildScopeFilter` uses `d.id IN (${Prisma.join(scope!.docIds!)})` (Phase 17.1 fix — DO NOT regress).

CTE structure (from D-21-03, slightly corrected — Postgres recursive CTE with depth tracking needs a depth column carried through the UNION):
```sql
WITH RECURSIVE neighbors(id, depth) AS (
  SELECT d.id, 0 FROM imported_doc d
   WHERE d.id = ANY($1::text[]) AND d."spaceId" = $2
  UNION
  SELECT dl."toDocId", n.depth + 1
    FROM doc_link dl JOIN neighbors n ON dl."fromDocId" = n.id
   WHERE n.depth < $3 AND dl."toDocId" IS NOT NULL
  UNION
  SELECT dl."fromDocId", n.depth + 1
    FROM doc_link dl JOIN neighbors n ON dl."toDocId" = n.id
   WHERE n.depth < $3
)
SELECT DISTINCT id FROM neighbors
```
Postgres requires `UNION` (not UNION ALL) to terminate cycles automatically — `UNION` dedups on (id, depth) which prevents infinite loops only because depth is monotonically increasing and bounded by $3; OK.

      spaceId: string,
      query: string,
      limit = 10,
      scope?: DocSearchScope,
      options?: { traverseLinks?: boolean; maxHops?: number }
    ): Promise<DocSearchResult[]> {
      let effectiveScope = scope;
      if (options?.traverseLinks && scope?.docIds && scope.docIds.length > 0) {
        const maxHops = options.maxHops ?? 2;
        if (maxHops < 1 || maxHops > 3) {

      }

    }
