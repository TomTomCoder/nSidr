<objective>
Implement ARH-05 — auto-trigger the embedding pipeline when a knowledge doc is created or updated via the agent MCP path (`create_knowledge_doc` / `update_knowledge_doc`). Today these calls flow through `KnowledgeDocService.createDoc` / `updateDoc`, which set `isIndexed=false` and rely on `DocIndexRecoveryService.onApplicationBootstrap()` to enqueue at boot — meaning docs are not searchable until the next server restart.

Purpose: Match the existing `doc-crud.controller.ts` enqueue pattern so docs are searchable seconds after agent creation.
Output: `KnowledgeDocService` injects `DOC_INGEST_QUEUE` and calls `enqueueReindex(docId, spaceId)` after every persistent write that affects `content`.
</objective>
