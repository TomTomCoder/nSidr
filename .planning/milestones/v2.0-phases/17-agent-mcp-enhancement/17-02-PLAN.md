<objective>
Wave 1 (AGENT-05). Make the agent an **MCP client**. Add an `AgentMcpServer` schema row type, an aggregator that uses the SDK `Client` to connect to (a) Teable's own MCP server (17-01) and (b) external MCP servers, adapts their tools via the 17-00 adapter into `ToolDefinition[]`, and merges them into `getToolsForAgent`. The execution loop is unchanged except that the `executeToolCall` default branch now dispatches MCP-namespaced tool calls to the aggregator's executor.

Output: new schema model + migration, aggregator service, registry merge, execution dispatch branch.
NOTE: 17-CONTEXT named this model `AgentConnection`, but `AgentConnection` ALREADY EXISTS for OAuth providers (token/scopes). Use a NEW `AgentMcpServer` model — do not overload AgentConnection.
</objective>
