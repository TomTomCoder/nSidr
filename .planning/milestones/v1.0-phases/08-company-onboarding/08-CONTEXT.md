# Phase 8: Company Onboarding - Context

**Gathered:** 2026-05-15
**Status:** Ready for planning
**Depends on phases:** 04 (agents), 06 (OAuth integrations), 07 (doc search)

<domain>
## Phase Boundary

Phase 8 delivers an "Import your company" onboarding wizard that transforms raw business data (Excel, CSV, PDF) into a fully configured Teable workspace: tables, automations, agents, and integrations — all generated by AI from the uploaded context.

What this phase builds:
- 5-step onboarding wizard UI (modal stepper)
- `CompanyProfile` Prisma model storing wizard state + uploaded file metadata
- File upload pipeline reusing existing `import` feature (Excel/CSV) and Phase 7 doc ingest (PDF)
- AI column mapper: detects column semantics from imported files → proposes Teable field types + table names
- AI context analyzer: reads all uploaded documents + mapped tables → proposes automations + agents
- Integration picker: OAuth connect flow for Gmail, Google Calendar, Slack (Phase 6 `OAuthIntegrationModule`)
- Onboarding summary page: shows what was created (tables, automations, agents, connections)
- Async processing via `COMPANY_ONBOARDING_QUEUE` (BullMQ)
- New page: `apps/nextjs-app/src/pages/base/[baseId]/onboarding.tsx`

**Scope boundary:** This phase ORCHESTRATES existing phase outputs — it creates no new AI capabilities, no new import logic, and no new OAuth flows. It assembles Phase 4 + 6 + 7 into a guided UX.
</domain>

<decisions>
## Implementation Decisions

### Prisma Model

- **D-01:** New `CompanyProfile` model in schema:
  ```prisma
  model CompanyProfile {
    id               String                  @id @default(cuid())
    spaceId          String
    space            Space                   @relation(fields: [spaceId], references: [id], onDelete: Cascade)
    companyName      String
    industry         String?
    employeeCount    Int?
    onboardingStatus CompanyOnboardingStatus @default(PENDING)
    importedFiles    Json                    @default("[]")  // array of { name, type, size, docId? }
    analysisResult   Json?                   // AI-generated proposals (tables, automations, agents)
    createdBy        String
    creator          User                    @relation(fields: [createdBy], references: [id])
    createdAt        DateTime                @default(now())
    updatedAt        DateTime                @updatedAt

    @@index([spaceId])
    @@unique([spaceId])  // one onboarding profile per space
  }

  enum CompanyOnboardingStatus {
    PENDING
    IMPORTING
    ANALYZING
    READY
    COMPLETE
  }
  ```
- **D-02:** `@@unique([spaceId])` — one active onboarding profile per workspace. Re-running wizard updates the existing row.

### Wizard Flow — 5 Steps

- **D-03:** Five locked steps, stored as `onboardingStatus` enum:

  | Step | Name | Status value | What happens |
  |------|------|-------------|-------------|
  | 1 | Company Info | PENDING | Name, industry, employee count |
  | 2 | Import Files | IMPORTING | Upload Excel/CSV/PDF (multi-file, drag-drop) |
  | 3 | AI Analysis | ANALYZING | Background job; polling spinner |
  | 4 | Connect Integrations | READY | OAuth connect for Gmail / Google Calendar / Slack |
  | 5 | Review & Activate | COMPLETE | Summary + "Create everything" button |

- **D-04:** Wizard persists state via `PATCH /api/spaces/:spaceId/onboarding` after each step. If user leaves and returns, wizard resumes at `onboardingStatus` step. "Create everything" button in step 5 is the only action that commits tables/agents/automations to the workspace.

### File Upload (Step 2)

- **D-05:** Supported formats: `.xlsx`, `.xls`, `.csv` (handled by existing `import` feature in `apps/nestjs-backend/src/features/import/`), `.pdf` (handled by Phase 7 `DocIngestionService`). Max 50 MB per file, max 20 files.
- **D-06:** Excel/CSV files are parsed immediately (sync) to extract column names + sample rows (first 5 rows). Result stored in `importedFiles` JSON. PDF files are queued via `DOC_INGEST` queue (Phase 7) and `docId` is stored back in `importedFiles` entry.
- **D-07:** File upload endpoint: `POST /api/spaces/:spaceId/onboarding/files` (multipart). Responds with `{ fileId, columns?: string[], sampleRows?: object[], docId?: string }` depending on type.

### AI Column Mapper (Step 3, part 1)

- **D-08:** Uses `PromptService.get('import.analyze', default)` (Phase 2 key already exists). Sends column names + 5 sample rows per file to the LLM. Receives proposed Teable field types (`text`, `number`, `date`, `singleSelect`, `multipleSelect`, `checkbox`, `url`) + a suggested table name.
- **D-09:** Mapping result stored in `analysisResult.tables` array: `[{ suggestedName, columns: [{ name, type, options? }] }]`. Never auto-creates tables — proposals only shown in step 5 UI.
- **D-10:** Queue: `COMPANY_ONBOARDING_QUEUE` (BullMQ). Job dispatched after file upload completes. Polling endpoint: `GET /api/spaces/:spaceId/onboarding` — frontend polls every 3 s while `onboardingStatus === 'ANALYZING'`.

### AI Context Analyzer (Step 3, part 2)

- **D-11:** After column mapping, a second LLM call reads: table proposals + PDF doc summaries (from Phase 7 `ImportedDoc.title` + first chunk content). Proposes:
  - Up to 3 automations (name + trigger type + action description)
  - Up to 2 agents (name + instructions + suggested tools from Phase 4's 5 built-in tools)
- **D-12:** Results stored in `analysisResult.automations` and `analysisResult.agents`. These are text proposals — no workflow/agent objects are created yet.
- **D-13:** LLM prompt key: `onboarding.analyze` — new key added to `PROMPT_KEY` in Phase 2's `prompt.service.ts`. Default prompt instructs the LLM to output JSON matching the `analysisResult` shape.

### Integration Picker (Step 4)

- **D-14:** Reuses Phase 6 `OAuthIntegrationModule` entirely. The integration picker UI is a simplified `IntegrationsPanel` showing only Gmail, Google Calendar, and Slack (the 3 most commonly needed for business workflows). Other providers available via full settings panel.
- **D-15:** OAuth popup pattern identical to Phase 6 (parent window listens for `postMessage { type: 'oauth_success' }`). No new OAuth code needed.

### Activation (Step 5)

- **D-16:** "Create everything" button triggers `POST /api/spaces/:spaceId/onboarding/activate`. The backend:
  1. Creates tables from `analysisResult.tables` via existing table creation API (calls `TableService.createTable()` for each)
  2. Creates agents from `analysisResult.agents` via `AgentService.create()` (Phase 4)
  3. Sets `onboardingStatus = COMPLETE`
  4. Returns created resource IDs
- **D-17:** Automation proposals are shown as text descriptions only — users create automations manually from the summary. Full automation creation via AI is out of scope (existing `workflow` feature handles this separately via "Create automation with AI").
- **D-18:** Activation is synchronous (no queue) — table + agent creation is fast enough (<2 s for typical wizard results). If more than 10 table proposals exist, batch them with a 100 ms delay between each to avoid overloading the event system.

### Module Architecture

- **D-19:** New `OnboardingModule` at `apps/nestjs-backend/src/features/onboarding/`. Imports: `ImportModule` (existing), `DocSearchModule` (Phase 7), `AgentModule` (Phase 4), `OAuthIntegrationModule` (Phase 6), `AiModule`. Registered in `app.module.ts`.
- **D-20:** `COMPANY_ONBOARDING_QUEUE = 'company-onboarding'` — registered via `EventJobModule.registerQueue(COMPANY_ONBOARDING_QUEUE)` following Phase 3 pattern.
- **D-21:** New prompt key `onboarding.analyze` added to `PROMPT_KEY` object in Phase 2's `prompt.service.ts`. Default: French-language prompt that outputs JSON `{ tables: [...], automations: [...], agents: [...] }`.

### UI Architecture

- **D-22:** Page route: `apps/nextjs-app/src/pages/base/[baseId]/onboarding.tsx`. Accessible from the base sidebar (new "Onboarding" item, only visible if `onboardingStatus !== 'COMPLETE'` or within 7 days of completion).
- **D-23:** Wizard uses a `<Stepper>` pattern with 5 `<Step>` components. Follow the 3-step agent wizard pattern from Phase 4 (React context for state, modal overlay, step progress indicator "ÉTAPE X SUR 5").
- **D-24:** File upload UI: drag-and-drop zone + file list with type icons. Shows progress bar per file during upload. Remove button per file.
- **D-25:** Step 3 (AI Analysis) shows animated spinner with status text (e.g., "Analyse des colonnes...", "Génération des propositions..."). Polling via `useQuery` with `refetchInterval: 3000` while `status === 'ANALYZING'`.
- **D-26:** Step 5 (Review): three collapsible sections — "Tables à créer" (table list with column previews), "Automatisations suggérées" (text cards, read-only), "Agents à créer" (agent cards with tool badges). User can remove items before activating.

### Claude's Discretion

- Exact sidebar file path for the Onboarding navigation item
- Whether to show onboarding as a modal wizard or a full page (follow existing pattern in codebase)
- Column type confidence score display (show/hide percentage in Step 3 preview)
- Whether `analysisResult` should be normalized into separate DB columns vs staying as JSON (JSON is fine at this scale)

</decisions>

<code_context>
## Existing Code Insights

### Key Reuse Points
- **Excel/CSV import**: `apps/nestjs-backend/src/features/import/` already handles file upload + column detection. `OnboardingService` calls this service directly rather than re-implementing.
- **PDF ingestion**: Phase 7's `DocIngestionService.ingestMarkdown()` — call with extracted text from PDF. Reuse `pdf-parse` already added in Phase 7.
- **Table creation**: `TableService.createTable(baseId, { name, fields })` — existing API; `fields` shape matches the column mapper output.
- **Agent creation**: Phase 4 `AgentService.create(spaceId, dto)` — call with name + instructions from `analysisResult.agents`.
- **OAuth connect**: Phase 6 `OAuthIntegrationController` handles the full OAuth popup flow — the onboarding wizard just embeds the same "Connect" buttons.

### Integration Points
- `PromptService` (Phase 2): inject into `OnboardingAnalysisService`; call `get('onboarding.analyze', defaultPrompt)` for the AI analysis step
- `AiService` (Phase 4 context): use for the actual `generateText()` call in `OnboardingAnalysisService`
- `EventJobModule.registerQueue(COMPANY_ONBOARDING_QUEUE)` in `OnboardingModule.imports`
</code_context>

<testing>
## Testing Strategy

### Gate rule
`npx vitest run` + `npx tsc --noEmit` before each wave. Wave 4 (wizard UI) requires Playwright E2E.

### Unit Tests (Vitest)
- `onboarding-analysis.service.spec.ts` — mock `AiService.generateText`; test JSON parsing of LLM response into `analysisResult` shape; test fallback when LLM returns malformed JSON (empty proposals, no crash)
- `onboarding.service.spec.ts` — mock `TableService.createTable` + `AgentService.create`; test activation creates correct number of tables and agents from `analysisResult`; test `@@unique([spaceId])` upsert behavior
- File upload handler: mock `ImportService`; test Excel → column extraction; test PDF → `DOC_INGEST` queue dispatch

### Integration Tests (Vitest + test DB)
- `POST /api/spaces/:spaceId/onboarding` → 201 with `CompanyProfile` row
- `PATCH /api/spaces/:spaceId/onboarding` step 1 → status updates to `IMPORTING`
- `POST /api/spaces/:spaceId/onboarding/files` with CSV → 200 with `columns` array
- `GET /api/spaces/:spaceId/onboarding` while `ANALYZING` → 200 with `onboardingStatus: 'ANALYZING'`
- `POST /api/spaces/:spaceId/onboarding/activate` → 200; tables and agents created; `onboardingStatus: 'COMPLETE'`

### E2E Tests (Playwright — Wave 4)
- Complete full 5-step wizard with a sample CSV + PDF → verify summary shows created table + agent
- Upload invalid file type → error message shown (no crash)
- Connect Gmail in step 4 → OAuth popup → verify "Connected" badge appears
- Click "Create everything" → navigate to created table → verify columns match uploaded CSV

### What NOT to test
- LLM response quality (mocked in all tests)
- Phase 6 OAuth token exchange (tested in Phase 6)
- Phase 7 PDF text extraction (tested in Phase 7)
- Phase 4 agent execution runtime (tested in Phase 4)
</testing>

<deferred>
## Deferred Ideas

- **Automation creation from proposals**: Step 5 shows automation text proposals but doesn't auto-create them — users use existing "Create automation with AI" flow. Full automation creation from onboarding context deferred.
- **Re-run wizard**: After `COMPLETE`, re-running the wizard (adding more files, re-analyzing) deferred — delete the `CompanyProfile` row to reset.
- **Multi-workspace onboarding**: One `CompanyProfile` per space only. Cross-workspace templates deferred.
- **Onboarding analytics**: Track completion rate, drop-off step, most-used integrations — deferred to a future analytics phase.
- **Industry-specific templates**: Pre-populated field types and agent templates per industry (e.g., "Legal", "Marketing") — deferred.
</deferred>
