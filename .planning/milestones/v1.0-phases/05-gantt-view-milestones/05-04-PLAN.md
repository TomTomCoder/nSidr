    --- hooks/useGanttDrag.ts ---
    Three drag behaviors:

    1. Bar move (drag bar body):
    - onMouseDown on bar body: record dragStartX = e.clientX, original startDate/endDate.
    - onMouseMove (window-level): compute deltaX = e.clientX - dragStartX.
       newStartDate = pixelToDate(dateToPixel(originalStart) + deltaX, timelineStart, timeScale)
       newEndDate = pixelToDate(dateToPixel(originalEnd) + deltaX, timelineStart, timeScale)
       Update local state to show a "ghost" position (no API call yet).
    - onMouseUp: call useUpdateRecord(tableId, recordId, { [startField]: newStartDate, [endField]: newEndDate })
       Clear drag state.

    2. Resize (drag left/right edge handles):
    - Left edge: only newStartDate changes. endDate stays fixed.
    - Right edge: only newEndDate changes. startDate stays fixed.
    - Same ghost-on-move, API-call-on-mouseup pattern.
    - Prevent start > end: clamp startDate to max(newStartDate, endDate - 1 day).

    3. Dependency creation (drag from right connector dot):
    - Render a small dot (8x8, rounded, border) at the right edge of each bar.
    - onMouseDown on dot: start "drawing" mode. Show a rubber-band SVG line following the mouse.
    - onMouseUp over another bar: call the dependency creation logic.
      If dependencyField is a text field: update that field as comma-separated with the target recordId appended.
      If dependencyField is a linked-record field: use the linked record API.
    - onMouseUp not over a bar: cancel.

    Export useGanttDrag hook: accepts { bars, ganttOptions, tableId, timelineStart, timeScale }
    Returns: { dragHandlers: { onMouseDown, onMouseMove, onMouseUp }, pendingGhostBars: GanttBarItem[] }

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| drag event → record update API | Mouse coordinates converted to dates and written back to records |
| options panel → view update API | User-selected fieldIds written to view.options |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-11 | T (Tampering) | drag-to-reschedule date computation | mitigate | Clamp computed dates to reasonable bounds (year 1900–2100) before calling updateRecord. Prevent startDate > endDate via clamp in resize handler |
| T-05-12 | E (Elevation of Privilege) | GanttOptionsPanel field update | accept | useUpdateView uses the same auth context as all other view mutations — no new privilege surface. Caller must have view-edit permission already enforced by backend |
| T-05-13 | T (Tampering) | dependency creation via text field | mitigate | Appended record IDs are validated against the current record set before being written. Cannot inject arbitrary text via the UI drag gesture |
| T-05-14 | D (Denial of Service) | window-level mousemove listener | accept | Listener is removed on mouseup and component unmount via cleanup in useEffect. No persistent leak |
</threat_model>
