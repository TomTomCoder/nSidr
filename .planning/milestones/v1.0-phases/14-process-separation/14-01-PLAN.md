<objective>
Fix the SSR axios baseURL bug and document the NestJS API-only env block.

Purpose: The current axios.ts uses `process.env.PORT` which is the Next.js process port, not the NestJS port. In separated mode this causes all SSR data fetches to hit the wrong process. This is the blocking prerequisite for process separation.

Output: A corrected axios.ts, a .env.development file with BACKEND_URL, and an annotated nestjs-backend/.env.
</objective>
