# Phase 1 — UI Design Contract: Authority Matrix

> Visual and interaction contract for the Authority Matrix management UI.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (via `@teable/ui-lib/shadcn`) |
| Preset | Teable CSS variable theme (global.shadcn.css) — light + dark supported |
| Component library | Radix UI (via shadcn wrappers in `@teable/ui-lib/shadcn`) |

## Layout

The page fills the `BaseLayout` main content area. Follow the `SettingRight` shell pattern:
- Header row: `border-b px-8 py-4` — title left, primary CTA right
- Content area: `flex-1 overflow-y-auto px-8 py-4`
- Matrix cards stack vertically with `gap-4` between them

---

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, checkbox-to-label gap |
| sm | 8px | Button inner padding, badge padding |
| md | 16px | Card internal padding, form field gaps |
| lg | 24px | Section padding (header `py-4` = 16px top+bottom, content `py-4`) |
| xl | 32px | Left/right page padding (`px-8` = 32px) |
| 2xl | 48px | Major vertical break between matrix cards |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Checkbox touch target: 20px (`size-5`) — standard Radix checkbox size
- Icon buttons: 28px minimum (`h-7 w-7`) for edit/delete actions in card header
- Role row height: 36px minimum (`h-9`) for inline-editable name input

## Color

All values from CSS custom properties — theme-aware (light + dark).

| Role | CSS Token | Hex (light) | Usage |
|------|-----------|-------------|-------|
| Dominant (60%) | `--background` | `#ffffff` | Page background, card background |
| Secondary (30%) | `--card` / `--secondary` | `#f4f4f6` | Matrix cards (`Card` component), role row backgrounds |
| Accent (10%) | `--primary` | `#1a2030` | Primary action button ("+ Nouvelle matrice"), "Enregistrer" buttons |
| Destructive | `--destructive` | `hsl(0 84.2% 60.2%)` | Delete matrix button, delete role button, confirmation dialog CTA |

Accent reserved for: the main "Nouvelle matrice" button in page header, and "Enregistrer" / "Ajouter un rôle" confirm buttons only. NOT used for every interactive element.

Borders: `border` class (uses `--border` = 8% opacity black/white) for card outlines and table row separators.
Muted text: `text-muted-foreground` for descriptions and placeholder text.

## Component Inventory

### Page Shell
```
<div className="h-full flex-col md:flex">
  <div className="flex items-center justify-between border-b px-8 py-4">
    <h2 className="text-2xl font-semibold tracking-tight">Matrice d'autorité</h2>
    <Button size="sm"><Plus className="size-4 mr-2" />Nouvelle matrice</Button>
  </div>
  <div className="flex-1 overflow-y-auto px-8 py-4 space-y-4">
    {/* MatrixCard list */}
  </div>
</div>
```

### MatrixCard
- Uses `Card` with `shadow-none border` and `CardContent className="p-4"`
- Header row: matrix name (editable via inline input or dialog) + description + Edit icon button + Delete icon button
- Roles section below a `border-t mt-4 pt-4` divider
- Roles listed as rows in a scrollable table-like grid
- Footer: `<Button size="xs" variant="outline"><Plus />Ajouter un rôle</Button>`

### Role Row
- Inline-editable name: `<Input className="h-7 text-xs w-32" />` — saves on blur
- Action checkboxes: grouped by category (see Action Grouping below), rendered as a horizontal scroll area
- Delete button: `<Button size="xs" variant="ghost" className="text-destructive hover:text-destructive">` with `<Trash2 className="size-3.5" />`

### Create/Edit Matrix Dialog
- `Dialog` with `DialogContent className="sm:max-w-md"`
- Fields: Name (required, `Input`), Description (optional, `Input`)
- Footer: Cancel + Enregistrer buttons

### Action Checkbox Grid
- Actions from `@teable/core Action` enum, grouped by category prefix
- Group header: `<p className="text-xs font-semibold text-muted-foreground mb-1">` with category label
- Each action: `<label className="flex items-center gap-1 text-xs"><Checkbox />{actionLabel}</label>`
- Groups rendered in a `flex flex-wrap gap-x-6 gap-y-3` container
- Overflow: horizontally scrollable per role row (`overflow-x-auto`)

### Action Category Grouping (Claude's discretion — from Action enum)
Group actions by the resource prefix in their name. Suggested groups:
| Category Label | Action prefix pattern |
|---|---|
| Tables | table:* |
| Champs | field:* |
| Vues | view:* |
| Enregistrements | record:* |
| Collaboration | collaborator:* |
| Base | base:* |
| Automatisations | automation:* |
| Autres | (any remaining) |

Category labels displayed in French to match the "Matrice d'autorité" heading locale.

### Loading State
Use skeleton placeholders (not spinner) for the matrix list on initial load:
- 2 skeleton cards: `<div className="h-40 rounded-lg bg-muted animate-pulse" />`
- Reason: page-level load; skeleton preserves layout, reduces perceived wait

### Empty State (no matrices yet)
- Centered in content area with `flex items-center justify-center`
- No `Alert` component — use plain text stack:
  - Heading: `text-sm font-semibold` — "Aucune matrice configurée"
  - Body: `text-xs text-muted-foreground` — "Créez votre première matrice pour définir des rôles personnalisés."
  - CTA button: same "Nouvelle matrice" button

## Interaction Contracts

| Interaction | Behavior |
|---|---|
| Create matrix | Click "+ Nouvelle matrice" → Dialog opens → fill name + description → "Enregistrer" → POST API → invalidate list → dialog closes → new card appears |
| Edit matrix name/description | Click Edit icon on MatrixCard header → same Dialog opens pre-filled → "Enregistrer" → PATCH API |
| Delete matrix | Click Trash icon → AlertDialog confirmation → "Supprimer" → DELETE API → card disappears |
| Add role | Click "+ Ajouter un rôle" → new empty role row inserted inline → name input focused → user types name → blur saves via POST API |
| Edit role name | Click role name input → edit inline → blur triggers PATCH API |
| Toggle action checkbox | Checkbox change → PATCH role API with updated actions array |
| Delete role | Click Trash on role row → immediate DELETE API call → row removed |
| Permission guard | `usePermission('base|authority_matrix_config')` → if false, render redirect or locked state (not 404) |

Optimistic updates: NOT used in this phase. Show loading state on mutation, then refetch. Keeps implementation simple and avoids rollback complexity.
