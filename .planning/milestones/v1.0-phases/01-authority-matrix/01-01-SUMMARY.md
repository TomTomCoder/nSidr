## Issues Encountered

**Task 3 — BLOCKED: No PostgreSQL connection available**

`pnpm prisma migrate deploy` failed with:
```
Error: Environment variable not found: PRISMA_DATABASE_URL.
```

Per plan instructions: "If `migrate deploy` fails due to missing DATABASE_URL or unreachable DB: Stop and report — do NOT mark task complete."

**To complete Task 3, the user must:**

1. Set the `PRISMA_DATABASE_URL` environment variable pointing to a running PostgreSQL instance, e.g.:
   ```bash
   export PRISMA_DATABASE_URL="postgresql://teable:teable@localhost:5432/teable?schema=public"
   ```
   Or add it to `apps/nextjs-app/.env.development` / `packages/db-main-prisma/.env`.

2. Apply the migration:
   ```bash
   cd packages/db-main-prisma && pnpm prisma migrate deploy --schema=prisma/postgres/schema.prisma
   ```

3. Regenerate the Prisma client:
   ```bash
   cd packages/db-main-prisma && pnpm prisma-generate
   ```

4. Verify the generated client contains the new models:
   ```bash
   grep -r "AuthorityMatrix" packages/db-main-prisma/node_modules/.prisma 2>/dev/null | head -5
   ```
