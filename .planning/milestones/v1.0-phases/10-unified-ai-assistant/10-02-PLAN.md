<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user message → LLM | User-supplied text enters LLM system prompt; must not leak workspace secrets |
| accept-proposal → DB write | Proposal acceptance triggers actual DB mutations |
| proposalId → message lookup | ProposalId is user-supplied in accept call; must be validated as belonging to the user's space |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-02 | Information Disclosure | workspace snapshot in system prompt | mitigate | Truncate snapshot to 8000 chars; include only id/name/type fields, not record data |
| T-10-03 | Elevation of Privilege | acceptProposal — user accepts proposal for another user's space | mitigate | Plan 03 controller validates spaceId ownership before calling acceptProposal; service validates proposalId belongs to a conversation in the caller's space |
| T-10-04 | Tampering | double-accept race condition | mitigate | accepted flag checked before execute; ConflictException returned on duplicate — per research Pitfall 3 |
| T-10-SC | Tampering | npm/pip/cargo installs | accept | No new packages in this plan |
</threat_model>
