## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP request → controller | spaceId in URL param is user-controlled; must not allow cross-space access |
| SSE stream → browser | Long-lived connection; client disconnect must be handled to avoid memory leak |
| GET conversations → DB | Returns only conversations for the authenticated user's space |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-05 | Elevation of Privilege | spaceId URL param — user could pass any spaceId | mitigate | JwtAuthGuard + space membership check (same pattern as existing space-scoped controllers); surface in summary if not implemented |
| T-10-06 | Denial of Service | SSE connection never closed on client disconnect | mitigate | Wrap for-await loop in req.on('close', () => { cleanup }) — set a flag and break the generator on client disconnect |
| T-10-10 | Elevation of Privilege | accept-proposal with proposalId from another space | mitigate | Controller verifies conversation.spaceId === req.params.spaceId before calling acceptProposal |
| T-10-SC | Tampering | npm/pip/cargo installs | accept | No new packages |
</threat_model>
