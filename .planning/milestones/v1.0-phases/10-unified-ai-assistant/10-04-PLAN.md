## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser fetch → POST /ai/chat | User message is the request body; SSE response may include workspace data |
| Accept button → POST /ai/accept-proposal | Frontend triggers write mutation; idempotency handled server-side |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-07 | Information Disclosure | workspace snapshot rendered in chat thread | accept | Snapshot belongs to the authenticated user's workspace; no cross-user data |
| T-10-08 | Tampering | ProposalCard Accept — proposalId from SSE event injected into POST body | mitigate | Server validates proposalId belongs to a conversation in the authenticated user's space (Plan 03 guard) |
| T-10-09 | Denial of Service | ReadableStream reader not cancelled on component unmount | mitigate | UnifiedChatContainer useEffect cleanup cancels the reader via reader.cancel() on unmount |
| T-10-SC | Tampering | npm/pip/cargo installs | accept | No new packages |
</threat_model>
