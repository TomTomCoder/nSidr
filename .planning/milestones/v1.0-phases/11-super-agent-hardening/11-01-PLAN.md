<objective>
Authorize every AgentController route. Today no route in `AgentController` (`@Controller('api/agent')`) carries a permission guard, so any authenticated user can run, edit, or delete any agent in any base. This plan adds a base-scoped guard that resolves `baseId` from the agent record and decorates each route with the correct `base|read` / `base|create` permission, mirroring `AiController`.

Purpose: Close the authorization gap (AC2). Prevent cross-base agent access.
Output: `AgentPermissionGuard` + decorated controller routes, registered in the module.
</objective>

| T-11-02 | Information Disclosure | cross-base agent read | mitigate | getResourceId returns agent.baseId so permission check is scoped to owning base |
| T-11-03 | Spoofing | :id/webhook | accept | webhook intentionally public; authenticated via X-Agent-Secret only (unchanged) |
