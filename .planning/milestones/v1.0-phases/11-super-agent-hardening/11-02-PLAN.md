<objective>
Make built-in tool/skill state persistent and honest (AC5). Today `AgentProfilePanel` initializes the 8 Teable tools to all-on cosmetically and only reads the `web_search` saved row; the backend `getToolsForAgent` only returns tools with an `isEnabled:true` row, so the UI and execution disagree. This plan threads the saved `agentTool` rows into the UI on load and aligns the execution default.

Purpose: Tool toggles reflect and persist real saved state across reload.
Output: UI initializes from the tools endpoint; registry default documented and consistent.
</objective>
