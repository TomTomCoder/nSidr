<objective>
Two coupled execution fixes (AC1 + AC3). (1) Multi-turn memory: `run()` always calls `createConversation` and seeds only a system+user message, so the agent never remembers earlier turns; thread the UI's `conversationId` through `POST :id/run` and load prior messages into the LLM message array. (2) Resilience: `streamLlmIteration` calls `aiService.getAIConfig`/`getModelInstance`, both of which throw `CustomHttpException` when AI/Gateway is unconfigured — this propagates out of the async generator and aborts the stream. Wrap model resolution so the failure surfaces as a structured `error` SSE event with an actionable message.

Purpose: Prove end-to-end execution and graceful failure; enable conversational follow-ups.
Output: conversationId-aware run loop with guarded model resolution.
</objective>
