  <action>
Apply a cache-aside pattern in each file. The pattern is:
```typescript
// READ
const cacheKey = `tableSchema:${tableId}` as const;
const cached = await this.cacheService.get(cacheKey);
if (cached) return cached;
const result = await this.prismaService.tableMeta.findUniqueOrThrow({ where: { id: tableId } });
await this.cacheService.setDetail(cacheKey, result, 300); // 300 s = 5 min
return result;

// WRITE / INVALIDATION
await this.prismaService.tableMeta.update({ ... });
await this.cacheService.del(`tableSchema:${tableId}`);
```

Specific locations:

**table.service.ts:**
- Find the method that fetches a single table's metadata by `tableId` (likely `getTable` or similar `findUniqueOrThrow` by id).
- Wrap with cache-aside using key `tableSchema:${tableId}`, TTL = 300 s.
- In any `update` or `delete` method for TableMeta, call `this.cacheService.del(`tableSchema:${tableId}`)` after the Prisma write.

**field.service.ts:**
- This file was already modified in Plan 01. Find methods that fetch fields for a table (key them as `tableSchema:fields:${tableId}`).
- TTL = 300 s. Invalidate on field create/update/delete.

**permission.service.ts:**
- Find the method that resolves permissions for a user+resource. Cache with key `userPermission:${userId}:${resourceType}:${resourceId}`, TTL = 120 s.
- Invalidate on permission grant/revoke events.

Do NOT cache inside transactions — only cache reads that happen outside of `this.prismaService.txClient()` calls. If the target method uses `txClient()`, skip caching for that specific call.

Inject `CacheService` into any service that doesn't already have it, following NestJS DI patterns (add to constructor, add to providers in module if missing).
