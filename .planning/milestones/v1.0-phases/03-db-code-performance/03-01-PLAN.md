  </read_first>
  <action>
In both service files, search for any pattern where a `findMany` (or `findFirst`) is called inside a `.map()`, `for...of`, or `forEach` loop iterating over records/fields/views.

For each N+1 found:
1. Replace the inner per-item query with a single upfront `findMany({ where: { id: { in: ids } } })` before the loop.
2. Build a `Map<string, Item>` keyed by `id` from the batch result.
3. Replace the inner call with a `map.get(item.id)` lookup.

Additionally, for any `field.findMany` or `view.findMany` that selects all columns but only uses `id`, `name`, `type`, or `tableId`, add an explicit `select` block limiting to those fields:
```typescript
select: {
  id: true,
  name: true,
  type: true,
  tableId: true,
  // add only the fields actually consumed downstream
}
```

Do NOT add `select` to queries where `fieldMeta`, `options`, or `lookupOptions` columns are actually consumed — those queries must stay unrestricted.
  </action>
  <verify>
    <automated>grep -n "findMany\|findFirst" apps/nestjs-backend/src/features/field/field.service.ts apps/nestjs-backend/src/features/view/view.service.ts | wc -l</automated>
  </verify>
  <acceptance_criteria>
    - No `findMany` or `findFirst` call appears inside a `.map(` or `for (` loop body in either file
    - At least one `select: {` block is present in field.service.ts
    - TypeScript compiles: `npx tsc --noEmit -p apps/nestjs-backend/tsconfig.json` exits 0
  </acceptance_criteria>
