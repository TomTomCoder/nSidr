## Layout Contract

**Inner split inside `SpaceInnerLayout` `{children}` slot — no new layout file (D-01).**

```
DocLibrary (h-full, flex, flex-row, overflow-hidden)
├── DocFolderTree panel
│     re-resizable: defaultWidth=240px, minWidth=160px, maxWidth=400px
│     enable={{ right: true }}
│     className: "flex h-full flex-col border-r bg-background"
└── DocEditorArea
      className: "flex flex-1 flex-col overflow-hidden"
```

The `Resizable` component from `re-resizable` wraps `DocFolderTree`. Pattern taken directly from `Sidebar.tsx`.

**Primary focal point declaration:** When a document is open, the CodeMirror editor content area is the primary visual anchor. The active tree node (`bg-accent` highlight) serves as the secondary anchor connecting the navigation to the content.

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`p-1`, `gap-1`) | Icon gaps, status dot margin |
| sm | 8px (`p-2`, `gap-2`) | Tree node inner padding, button icon gap |
| md | 16px (`p-4`, `px-4`) | Panel body padding, editor content padding |
| lg | 24px (`py-6`) | Empty state vertical rhythm |
| xl | 32px (`py-8`) | Empty state icon container padding |
| 2xl | 48px (`py-12`) | No-doc-selected empty state top margin |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Tree node row height: 32px (`h-8`) — matches `BaseNodeTree` item height
- Touch target minimum: 48px tall for toolbar mode-toggle buttons on touch (achieved via `min-h-12`)
- Panel header: `px-4 py-2` (16px / 8px) — matches `BaseSideBar` header compact density

## Typography

| Role | Size | Weight | Line Height | Tailwind Classes |
|------|------|--------|-------------|-----------------|
| Body / doc content | 14px | 400 (regular) | 1.5 | `text-sm` |
| Label / tree node name | 13px | 400 (regular) | 1.4 | `text-xs` or `text-sm` |
| Panel heading | 14px | 600 (semibold) | 1.4 | `text-sm font-semibold` |
| Empty state heading | 14px | 600 (semibold) | 1.4 | `text-sm font-semibold` |

Rules:
- Exactly 2 weights used: 400 (regular) and 600 (semibold). No other weights.
- Truncated tree node names: `truncate` class, single line, no wrap.
- Editor title input: `text-lg font-semibold` (18px / 600) — the only 18px usage in this phase.
- Markdown preview body: `prose prose-sm dark:prose-invert` via Tailwind typography plugin (already in app).

## Color

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `bg-background` / `text-foreground` | Main surface, editor area, tree panel background |
| Secondary (30%) | `bg-muted` / `border` | Hover backgrounds, panel header, resize handle, separator line |
| Accent (10%) | `bg-primary` / `text-primary-foreground` | Reserved for specific elements only (see below) |
| Destructive | `text-destructive` / `bg-destructive` | Delete actions only |
| Success | `bg-success` / `text-success-foreground` | Indexed status dot only |
| Warning | `text-warning` | Pending-index status dot only |

**Accent reserved for:**
1. The "New Doc" primary CTA button in tree header
2. Active/selected tree node background (`bg-accent text-accent-foreground`)
3. Mode toggle active state (active mode button)

**Accent NOT used for:** hover states (use `hover:bg-muted`), folder chevrons, status dots, breadcrumb links.

### Index Status Dot Colors (D-05)

| State | Color | Class |
|-------|-------|-------|
| Pending (queued / processing) | Yellow | `bg-warning` (hsl(var(--warning))) |
| Indexed (isIndexed=true) | Green | `bg-success` (hsl(var(--success))) |
| Error | Red | `bg-destructive` (hsl(var(--destructive))) |

Dot size: `size-1.5` (6px circle), `rounded-full`, placed inline-right of doc node name.

## Component Inventory

### DocFolderTree Panel

**Header (top of panel):**
```
[Folder icon 14px] "Knowledge Base"  [+ New Folder btn] [Import btn]
```
- Height: `h-10` (40px)
- Classes: `flex items-center justify-between px-4 border-b`
- "New Folder" and "Import" buttons: ghost icon-only buttons (`Button variant="ghost" size="icon"`, `size-4` icon), with Tooltip

**DocFolder node:**
```
[ChevronRight 12px, rotates 90° when open] [Folder icon 14px] [name, truncated]  [...more button on hover]
```
- Row height: `h-8`
- Padding: `pl-{depth * 3 + 2} pr-2` (8px base indent per depth level + 8px right)
- Normal: `text-sm text-foreground`
- Hover: `hover:bg-muted rounded-sm`
- Selected/active: `bg-accent text-accent-foreground rounded-sm`
- ChevronRight: `transition-transform duration-150`, `rotate-90` when expanded
- "..." button (`MoreHorizontal` icon): `opacity-0 group-hover:opacity-100`, opens DropdownMenu

**ImportedDoc node:**
```
[File icon 14px] [name, truncated] [status dot 6px]  [...more button on hover]
```
- Row height: `h-8`
- Padding: `pl-{depth * 3 + 6} pr-2` (extra indent beyond parent folder)
- Normal: `text-sm text-foreground`
- Hover: `hover:bg-muted rounded-sm`
- Selected: `bg-accent text-accent-foreground rounded-sm font-semibold`
- Status dot: right-aligned, `ml-auto mr-1`, `size-1.5 rounded-full flex-shrink-0`

**Inline rename input:**
- Replaces the node label in-place
- Classes: `h-6 w-full rounded-sm border bg-background px-1 text-sm focus:outline-none focus:ring-1 focus:ring-ring`
- Enter = save, Escape = cancel
- Auto-select all text on mount

**Context menu (DropdownMenu):**
- "Rename" — `Pencil` icon 12px
- "New Document" (folders only) — `FilePlus` icon 12px
- separator
- "Delete" — `Trash2` icon 12px, `text-destructive focus:text-destructive`

**Folder tree scrollable body:**
- Classes: `flex-1 overflow-y-auto py-1 scrollbar-thin`
- No padding on sides — indent is per-node

### DocEditorArea

**Toolbar:**
```
[Breadcrumb: Folder > Doc name]    [mode toggle group: Edit | Split | Preview]  [status badge]
```
- Height: `h-10` (40px)
- Classes: `flex items-center justify-between border-b px-4 shrink-0`
- Breadcrumb: `text-xs text-muted-foreground` with `ChevronRight` 10px separators
- Mode toggle: `ToggleGroup` (shadcn) with 3 items. Active item: `bg-accent text-accent-foreground`
- Status badge: `Badge variant="outline"` — text "Indexing..." / "Indexed" / "Index failed"

**Title input (below toolbar, above editor):**
- Full-width text input, no border, no background
- Classes: `w-full bg-transparent px-4 py-3 text-lg font-semibold focus:outline-none placeholder:text-muted-foreground`
- Placeholder: "Untitled"
- Separator below: `border-b`

**CodeMirror container:**
- Edit mode: `flex-1 overflow-hidden` — CodeMirror fills entirely
- CodeMirror outer div: `h-full`
- Light mode: default CodeMirror theme
- Dark mode: `oneDark` from `@codemirror/theme-one-dark`
- `EditorView.lineWrapping` enabled (prose, not code)
- No line numbers (prose mode)
- Padding inside editor: 16px via CodeMirror `EditorView.theme({ ".cm-content": { padding: "16px" } })`

**Split mode layout:**
```
[CodeMirror — 50% width] [1px divider border-r] [react-markdown preview — 50% width]
```
- Container: `flex h-full overflow-hidden`
- Each pane: `w-1/2 overflow-y-auto`
- Preview pane classes: `prose prose-sm dark:prose-invert max-w-none px-6 py-4`

**Preview mode:**
- `react-markdown` only, full width
- Classes: `flex-1 overflow-y-auto`
- Inner: `prose prose-sm dark:prose-invert max-w-none px-6 py-4`

**Auto-save indicator:**
- Small transient text bottom-right of toolbar area or status bar
- Classes: `text-xs text-muted-foreground`
- "Saving..." → "Saved" → fades out after 2s

**No-doc-selected empty state:**
- Centered in `DocEditorArea`
- `FolderOpen` icon 48px, `text-muted-foreground opacity-30`
- Heading: `text-sm font-semibold text-muted-foreground`
- Body: `text-xs text-muted-foreground text-center max-w-xs`

### Delete Confirmation Dialogs

Uses `AlertDialog` from `@teable/ui-lib/shadcn`. Two distinct dialogs:

**Delete Document dialog:**
- Cancel button: `Button variant="outline"` — "Keep Document"
- Confirm button: `Button variant="destructive"` — "Delete"

**Delete Folder dialog:**
- Cancel button: `Button variant="outline"` — "Keep Folder"
- Confirm button: `Button variant="destructive"` — "Delete"

No third button in either dialog.

## Accessibility Notes

- Tree keyboard navigation: handled by `@headless-tree` `hotkeysCoreFeature` + `keyboardDragAndDropFeature` (D-02)
- Mode toggle: `ToggleGroup` uses Radix `role="group"` with `aria-label="Editor mode"`
- Status dots: pair each dot with `aria-label="Indexed"` / `aria-label="Indexing"` / `aria-label="Index failed"` on the span
- Inline rename input: `aria-label="Rename"` + `autoFocus` on mount
- Delete dialog: `AlertDialog` from shadcn provides correct focus trap and `role="alertdialog"`
- Resize handle: `aria-label="Resize folder panel"` on the re-resizable handle
