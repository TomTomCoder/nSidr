# Milestone v2.0 — AI Integrations: Requirements (Archived)

> Defined 2026-06-04. Archived 2026-06-14 at milestone completion. All requirements satisfied.
> REQ-IDs are per-category; phases continue from v1.0 (start at Phase 15).

## v2.0 Requirements — all delivered

### AI Provider Gateway (GW) — Vercel-AI-SDK gateway

- [x] **GW-01**: User can register an AI provider (OpenAI, Anthropic, Google, Mistral, DeepSeek, Ollama, …) with a **per-provider** API key. — Phase 15
- [x] **GW-02**: User can browse/search available models filtered by **provider** and **modality** (text / image / audio / video / embedding). — Phase 15
- [x] **GW-03**: User can run a **local model via Ollama** from within the app (launch + select). — Phase 15
- [x] **GW-04**: All AI features route through the gateway so model choice is provider-agnostic (single config swaps the model). — Phase 15 (embeddings via `UnifiedAiService.generateEmbeddings`)
- [x] **GW-05**: Per-model API-key override for cost-segregation. — Phase 15 (2-level cascade in `resolveApiKey`)

### AI-Generation Column (AICOL)

- [x] **AICOL-01**: AI field whose cell value is generated by a selected model from other columns. — Phase 16
- [x] **AICOL-02**: Output constrained by typology — free text, enum, number, JSON shape, classification — with validate-and-repair. — Phase 16 (`AiOutputValidationService`, Zod schemas)
- [x] **AICOL-03**: On-demand cell regeneration. — Phase 16 (regenerate endpoint + cell context menu)

### Agent System (AGENT) — MCP-based

- [x] **AGENT-01**: Agent reads/acts on **interfaces** (apps/dashboards). — Phase 17
- [x] **AGENT-02**: Agent invokes **installed plugins** as tools. — Phase 17
- [x] **AGENT-03**: Knowledge search scoped to user-selected docs/folders. — Phase 17
- [x] **AGENT-04**: Agent **retains memory** across conversations. — Phase 17 (verified existing `AgentMemoryService`)
- [x] **AGENT-05**: Agent tools exposed via **MCP**. — Phase 17

### Agent MCP Hardening (AGENT-LIVE) — Phase 17.1

- [x] **AGENT-01-LIVE**: Interface tools registered in MCP `tools/list` and dispatched.
- [x] **AGENT-05-LIVE-rbac**: MCP RBAC gate honours space-inherited roles via `PermissionService.getEffectiveBaseRole`.
- [x] **AGENT-05-LIVE-scope**: Deterministic vitest integration spec proves `search_knowledge_base` scope wiring.

### External Databases (EXTDB)

- [x] **EXTDB-01**: External VectorDB connector (Qdrant) as RAG source. — Phase 18 (`QdrantConnectorService` + hybridSearch RRF merge)
- [x] **EXTDB-02**: External Postgres for federated query (read-only virtual tables). — Phase 18 (`ExternalPgConnectorService` + `assertSelectOnly` guard)

### Extension System (EXT)

- [x] **EXT-01**: Spike — OpenClaw assessment, license compatibility, port-vs-reimplement decision. — Phase 19
- [x] **EXT-02**: Install third-party extensions (MCP/manifest, install-by-URL, consent). — Phase 19

### AI-Surface UI (UI)

- [x] **UI-01**: Simplified AI/model-picker + per-provider key management modal. — Phase 20 (`AiSettingsHub`, `UnifiedModelPicker`, `InlineProviderKeyList`)
- [x] **UI-02**: Simplified AI-column configuration UI. — Phase 20 (`CompactAiFieldConfig`)

### Knowledge Graph (KG) — Phase 21

- [x] **KG-01**: Agents create/update md docs via MCP; `imported_doc.sourceType='agent'`; doc-ingest worker picks up async embedding.
- [x] **KG-02**: Docs link via `doc_link` table; `link_docs` + `get_doc_links` MCP tools enforce same-space RBAC; read-only "Linked docs" sidebar.
- [x] **KG-03**: `search_knowledge_base` accepts optional `traverseLinks` + `maxHops` (1..3) using Postgres recursive CTE; default `false` (backward-compatible).

### Automations / RPA (AUTO) — Phase 22

- [x] **AUTO-01**: Agents list/read automations via `list_workflows`, `get_workflow` MCP tools with service-level RBAC.
- [x] **AUTO-02**: Agents trigger automations via `run_workflow` with input args; cross-base rejection with typed error envelope.

### Agent Runtime Hardening (ARH) — Phase 24

- [x] **ARH-01**: AI Gateway failover via `fallbackModels` retry loop in `streamLlmIteration`.
- [x] **ARH-02**: Input/Output Guardrails — `GuardrailService.validateWrite` rejects invalid writes before DB with structured error.
- [x] **ARH-03**: HITL — `request_human_approval` tool suspends run; `waiting_for_approval` status; approval card + accept/reject endpoints.
- [x] **ARH-04**: Per-user OAuth credential binding — `AgentConnection.userId` + `(agentId, userId, provider)` resolution with agent-level fallback.
- [x] **ARH-05**: Embedding auto-trigger — `create_knowledge_doc` / `update_knowledge_doc` enqueue BullMQ reindex job; doc searchable within seconds.

## UI/UX Replatform Spike (UIRE) — Phase 23

- [x] **UIRE-01**: Spike artifact `DECISION.md` — Option A (keep current UI + Phase 20 polish) selected with stakeholder sign-off; Option-A maintenance sprint shipped.

## Out of Scope (v2.0) — confirmed deferred

- Vercel AI Gateway / OpenRouter as the gateway — managed/SaaS-only; in-process Vercel-AI-SDK chosen for self-host.
- Full ground-up UI rebuild — incremental only (validated by Phase 23 spike).
- Importing external Postgres data into Teable tables — federated/virtual-table read only.
- Chat-channel connectors (WhatsApp/Telegram/etc. from OpenClaw) — not Teable's use case.
