## Milestone 2: Super Agent Production-Readiness

### Phase 11: Super Agent Hardening
**Goal:** Close the gap between the shipped Super Agent UI/trigger surface and a system that actually runs reliably, securely, and verifiably — prove end-to-end execution against a configured model, secure every agent endpoint with permission guards, give the agent real multi-turn conversation context, emit the missing DM trigger, make tool/skill state truly persistent, and add the first regression tests
**Depends on:** Phase 4, Phase 10
**Plans:** 3/6 plans executed

Plans:
- [x] 11-01-PLAN.md — Authorize every AgentController route via AgentPermissionGuard (baseId from agent record) [AC2]
- [x] 11-02-PLAN.md — Persistent built-in tool/skill state (UI load + registry default) [AC5]
- [x] 11-03-PLAN.md — Multi-turn conversation context + graceful AI-config error surfacing [AC3, AC1]
- [ ] 11-04-PLAN.md — POST :id/message DM emitter + human-verified live run [AC4, AC1 checkpoint]
- [ ] 11-05-PLAN.md — First regression tests: trigger CRUD, record-projection listener, webhook secret, execution happy path [AC6]
- [ ] 11-06-PLAN.md — (optional/stretch) Record tools via real field→column model [AC7]

Scope (gaps identified post-milestone, 2026-05-30):
1. **Execution proof + resilience** — verify `AgentExecutionService` runs end-to-end against a real model; graceful degradation + clear error surfacing when AI Gateway/base AI config is missing (currently throws an unhandled config error mid-run)
2. **Authorization** — add `@Permissions` guards to all ~20 `AgentController` endpoints (create/run/delete/triggers/tools/memories/conversations); scope agent access to the owning base
3. **Multi-turn context** — feed prior `AgentConversation` messages back into the LLM loop so chat has memory across turns (today every run is one-shot from system+user)
4. **DM trigger source** — emit `agent.dm` from the real DM/message path (handler exists, nothing fires it)
5. **Persistent tool/skill state** — load all built-in tool enablement from `agentTool` on mount (today only `web_search` is read; the 8 Teable tools are cosmetic-on)
6. **Regression tests** — first `*.spec.ts` coverage for trigger CRUD, the record→agent projection path, webhook auth, and the execution loop
7. **(stretch) Robust record tools** — replace raw `$queryRawUnsafe` JSON-column access in `create_record`/`search_records` with Teable's real field→column model
